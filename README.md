## Main features



* Reads FITS files and headers from a user-defined input directory.

* Applies instrument-specific masking.

* Uses header-based image dimensions for defining the Optical Flow ROI.

* Applies temporal GTF filtering for ASPIICS to mitigate brightness flickering.

* Rotates ASPIICS images to solar-north-up using SunPy (in `DOFCAT_preprocessing_ASPIICS.py`). Not added here deliberately for METIS (`DOFCAT_preprocessing_METIS.py`), so users can see the difference. The correct approach would be to add the roll correction to METIS as well.
  
* Creates running-difference images using frame `i+2 - i`, and preprocessing.

* Computes dense optical flow for METIS difference images using Farneback’s method

* Converts pixel motion to physical velocity in km/s

* Saves figures, headers, and compressed NumPy arrays in a clean output structure

## Scripts



### 1. `DOFCAT_preprocessing_METIS.py`



This script processes METIS Total Brightness FITS files and creates running-difference images.



#### What it does



1. Loads all FITS files from the input folder.

2. Reads the image data and headers.

3. Creates a valid field-of-view mask using instrument metadata.

4. Computes running differences using `I(i+2) - I(i)`.

5. Applies the mask to remove invalid pixels.

6. Computes plotting extent from WCS keywords.

7. Saves PNG images with consistent scaling.

8. Saves the aligned headers and compressed difference images.



#### Output



The script creates:



```text

save_dir/

└── difference_images/

    ├── difference_image_0000.png

    ├── difference_image_0001.png

    ├── ...

    ├── difference_headers.pkl

    └── diff_imgs.npz

```



#### Notes



* The figure size is derived from `NAXIS1` and `NAXIS2` in the FITS header.

* The output directory is user-defined.



---



### 2. `DOFCAT_preprocessing_ASPIICS.py`



This script processes PROBA-3 / ASPIICS Wideband Level 2 FITS files and creates filtered, rotated running-difference images.



#### What it does



1. Loads all FITS files from the input folder.

2. Computes running differences using `I(i+2) - I(i)`.

3. Stacks the difference images into a time cube.

4. Applies a temporal Gaussian Tapered Fourier (GTF) filter to reduce flickering.

5. Rotates each frame to solar north up using SunPy.

6. Applies an annular coronal mask.

7. Plots the processed images with a solar-disk overlay.

8. Saves the final PNGs, headers, and compressed NumPy arrays.



#### Output



The script creates:



```text

save_dir/

└── difference_images/

    ├── difference_image_0000.png

    ├── difference_image_0001.png

    ├── ...

    ├── difference_headers.pkl

    └── diff_imgs.npz

```



#### Notes



* The temporal GTF filter is preserved as part of the processing chain.

* Rotation and masking are applied after filtering.


---



### 3. `DOFCAT_OpticalFlowPipeline_METIS.py`



This script takes the running-difference images generated by the preprocessing stage and computes dense optical flow.



#### What it does



1. Loads the saved PNG difference images. Converted back to machine-readable data.

2. Applies bilateral filtering while preserving CME structure.

3. Defines a Region of Interest (ROI).

4. Computes dense optical flow using Farneback’s method.

5. Converts pixel motion into physical velocity using FITS metadata.

6. Filters unrealistic vectors using velocity thresholds.

7. Saves heatmaps with quiver arrows.

8. Saves vector overlays on the original frames.



#### Output



The script creates:



```text

output_dir/

├── intensity/

│   ├── frame_0000.png

│   ├── frame_0001.png

│   └── ...

└── frames_with_vectors/

    ├── frame_0000.png

    ├── frame_0001.png

    └── ...

```



#### Notes



* The optical-flow ROI is defined so that the padded border (which includes labels, titles, and ticks) is excluded.

## Step-by-step workflow



### METIS preprocessing



The METIS preprocessing pipeline follows these steps:



1. Read FITS files and headers.

2. Build a mask for the usable coronal field of view.

3. Compute `I(i+2) - I(i)` running differences.

4. Mask invalid pixels with `NaN`.

5. Determine plotting extent from WCS metadata.

6. Save difference images as PNGs.

7. Save headers and compressed arrays for later analysis.



---



### ASPIICS preprocessing



The ASPIICS preprocessing pipeline follows these steps:



1. Read FITS files and headers.

2. Compute running differences.

3. Build a 3D difference cube.

4. Apply a temporal GTF filter in the time direction.

5. Rotate each filtered frame to solar-north-up.

6. Apply an annular coronal mask.

7. Save PNGs, headers, and compressed arrays.



---



### Optical flow analysis



The optical-flow pipeline follows these steps:



1. Load the saved difference-image PNGs.

2. Apply denoising.

3. Extract only the actual data region by removing padding.

4. Compute dense Farneback optical flow between consecutive frames.

5. Convert pixel displacement into km/s using FITS header information.

6. Remove unphysical motion using thresholds.

7. Save velocity heatmaps and vector overlays.

## Important implementation details



* The METIS preprocessing code is generalized through `NAXIS1` and `NAXIS2` from the FITS header.

* The optical-flow script uses a padding-aware ROI so only the real image data is used.

* The ASPIICS preprocessing script adds the temporal GTF filter and rotation steps.

* The physical velocity conversion uses metadata from the FITS headers rather than image pixels alone for accuracy.
